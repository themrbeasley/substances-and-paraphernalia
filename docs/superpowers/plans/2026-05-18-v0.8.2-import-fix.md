# v0.8.2 — Import Fix (Plan)

**Status:** awaiting user approval on open questions before execution
**Date:** 2026-05-18
**Companion bug context:** PR 18 + PR 19 shipped as v0.8.1; module is fully invisible at runtime due to three stale named imports.

## 1. Root cause (confirmed)

PR 18 (v0.8.1 lifecycle refactor, commit `96b142f`) renamed `getWithdrawalMod` → `getWithdrawalDc` in [scripts/data/flag-schema.js](../../../scripts/data/flag-schema.js) but missed three callers. Native browser ES module loading throws `SyntaxError: ... does not provide an export named '<name>'` at link time on **any one of them**, killing [scripts/module.mjs](../../../scripts/module.mjs) before its `init` hook runs.

| File | Line | Broken import |
|---|---|---|
| `scripts/hooks/drag-to-inventory.js` | 27 | `getWithdrawalMod` |
| `scripts/ui/simulate-dose.js` | 23 | `getWithdrawalMod` |
| `scripts/module.mjs` | 22 | `processAbstainFailure` (deleted by PR 18 rewrite of `long-rest-abstain.js`) |

**Why CI missed it:** [eslint.config.mjs](../../../eslint.config.mjs) does not load the `import` plugin, so `import/named` is not enforced. Unit tests are pure-Node and never import these Foundry-coupled files. `npm run validate` only validates pack content.

**PR 19 is not the cause** — its `disabled`-attribute additions can't hide UI. The v0.8.1 tag is attached to the PR 19 merge commit (`279be6f`), which is why the break was attributed there.

### Step 2 — patch [scripts/hooks/drag-to-inventory.js](../../../scripts/hooks/drag-to-inventory.js)

Two call sites (`CHOICES.ADDICTED` ~line 192, `CHOICES.WITHDRAWING` ~line 215) use `getWithdrawalMod` + `computeRestsRemaining` to seed `restsRemaining` on the actor flag at drop time. Per v0.8.1 spec §13.2, the actor-side `restsRemaining` field is gone — replaced by `{appliedAt, endsAt}`.

- Imports: drop `getWithdrawalMod`; drop `computeRestsRemaining` from `withdrawal.js`. Add `getWithdrawalDuration` from `flag-schema.js`. Add `durationToSeconds` from `withdrawal-duration.js`.
- Both branches: compute `endsAt = new Date(Date.now() + durationToSeconds(duration.value, duration.unit) * 1000).toISOString()`. Pass `{ appliedAt, endsAt }` to `setActorWithdrawalEntry`.
- For `CHOICES.WITHDRAWING`: clone withdrawal AE templates onto the actor as well, using the same pattern as `applyWithdrawalAeFromTemplate` in [scripts/hooks/long-rest-abstain.js](../../../scripts/hooks/long-rest-abstain.js) (lines ~198–228). With `restsRemaining` gone, Times-Up needs an AE with `duration.seconds` to expire — a flag-only entry has no expiry. Reuse `applyWithdrawalEffect` if exported from `addiction.js`; else extract a shared helper into a new module under `scripts/data/`.
- Update chat message format strings: `rests` arg disappears. Substitute with a human-readable duration (e.g. `"3 days"`) or revise the i18n key. Edit `lang/en.json` accordingly.
- Remove the now-unused `conMod(actor)` helper.

### Step 3 — patch [scripts/ui/simulate-dose.js](../../../scripts/ui/simulate-dose.js)

`preSeedAddictionState` (~line 358) uses the same recipe to seed a test actor's withdrawal flag for the simulate-dose tool.

- Imports: drop `getWithdrawalMod`; drop `computeRestsRemaining` import. Add `getWithdrawalDuration`, `durationToSeconds`.
- Body: same `endsAt`-from-duration logic. Write `{ appliedAt, endsAt }`.
- Function signature: drop `conMod` parameter (no longer used for math). Update the caller in `runSimulation` (line ~255) accordingly.
