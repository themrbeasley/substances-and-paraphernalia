# v0.8.4 — follow-ups from v0.8.3 hotfix

**Status:** Approved 2026-05-18.

## Why

Three loose ends surfaced while shipping the v0.8.3 hotfix. Bundling them
into v0.8.4 cleans up technical debt before any v0.9 feature work begins.

1. **Stale Quench suite.** `test/quench/test-suite.mjs` (~1,500 LOC) still
   references the removed `applyOrIncrementToleranceStack` API and the
   v0.8.1-deprecated flag shape (`withdrawalMod`, `restsRemaining`). The
   user has confirmed they no longer run Quench in a live world, so the
   entire `test/quench/` directory (14 files, ~3,600 LOC) is dead weight.
2. **Deprecated GitHub Actions.** `actions/checkout@v4` and
   `actions/setup-node@v4` bundle the Node-20 action runtime that GitHub
   has flagged as deprecated effective June 2026.
3. **Pre-existing lint warnings.** Five `no-unused-vars` warnings have
   accumulated across `addiction.js`, `abstain-dialog.js`, and
   `phase1-addiction-only.test.mjs`.

## Scope

**In:**
- Delete `test/quench/` and all references to it.
- Bump deprecated workflow actions to their current major.
- Remove the unused-vars warnings.

**Out:**
- Bumping `node-version: 20` in workflow setup-node steps. That's our
  *build runtime*, not the *GitHub Actions runtime* that GitHub has
  deprecated. Separate concern.
- Enforcing `--max-warnings 0` in the lint script. Considered and rejected
  for v0.8.4 — out of scope unless raised again.
- Any refactor, schema change, or new test beyond what these three items
  require.

## Approach

### Quench removal

Delete `test/quench/` (14 files). Update every reference to it:

| File | Change |
|---|---|
| `test/quench/` | Delete entire directory |
| `scripts/module.mjs` | Remove `registerQuenchSuiteIfActive()` function and its call site in `Hooks.once("init", …)` |
| `eslint.config.mjs` | Remove `Quench: "readonly"` from the `foundryGlobals` declaration |
| `module.json` | Remove `quench` from `relationships.recommends` if present |
| `CLAUDE.md` | Edit only the two sentences that become incorrect — the "Quench integration suite at `test/quench/test-suite.mjs`" paragraph and the "Quench tests are the canonical consumer" line in the Public API section. Don't touch surrounding architecture text. |
| `package.json` | No change; `test:unit` never referenced `test/quench/*` |

The Quench loader in `module.mjs` is the only runtime consumer. Removing
it in the same commit as the directory delete means there is no
intermediate broken state.

### Workflow action bumps

In both `.github/workflows/ci.yml` and `.github/workflows/release.yml`:
- `actions/checkout@v4` → `@v5`
- `actions/setup-node@v4` → `@v5`

Verify `softprops/action-gh-release@v2` is still the current major in
release.yml; if a v3 has shipped, bump that too. Otherwise leave alone.

### Unused-vars cleanup

Verify each import is truly unreferenced (the v0.8.1 refactor moved things
around — want to be sure nothing's about to call them), then remove from
the import statement:

- `scripts/hooks/addiction.js`: `getModifier`, `getToleranceDecay`,
  `clearActorToleranceEntry`
- `scripts/ui/abstain-dialog.js`: `MODULE_ID`

The fifth warning (`MODULE_ID` in `phase1-addiction-only.test.mjs`)
disappears automatically when the Quench directory is deleted.

## Execution order

One PR, branch `chore/v0.8.4-followups`, three commits in this order so
each verification step is meaningful:

1. `chore: delete Quench test suite and references`
2. `chore: bump deprecated GitHub Actions to v5`
3. `chore: remove unused imports`

Squash-merge to main, tag `v0.8.4`, release workflow handles the rest.

## Verification

After each commit, all of:

- `npm run lint` — by the end of commit 3, **0 warnings** (the proof we
  actually cleaned up, not just 0 errors)
- `npm run validate` — manifest + content
- `npm run test:unit` — should remain at 275 passing
- `npm run pack` — all four packs rebuild

## Risks

- **Quench delete:** zero runtime risk. The loader is the only consumer
  and it is deleted in the same commit.
- **Action bumps:** v5 of both `checkout` and `setup-node` are GA and have
  been in widespread production use for months.
- **Unused-vars:** zero risk by definition (removing code nothing
  references).

## Out-of-scope items considered

- **Enforce `--max-warnings 0` on the lint script.** Would prevent the
  warning count from silently drifting back up. Paid for by this PR
  reaching 0 warnings, so the marginal cost is one config line. Rejected
  for v0.8.4 to keep scope tight; revisit if drift recurs.
- **Bump `node-version: 20` to `22` in workflow setup-node steps.** Node
  20 reaches end-of-life in April 2026. Separate concern from the actions
  deprecation; would invalidate the `"engines": { "node": ">=20" }` line
  in `package.json` and warrants its own discussion.
