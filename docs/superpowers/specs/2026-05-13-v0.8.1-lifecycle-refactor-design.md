# v0.8.1 — Lifecycle Mechanics Refactor (Design)

**Status:** draft, awaiting review
**Authoritative source:** [`corrected-lifecycle-v6.html`](../../../.superpowers/brainstorm/312-1778682818/content/corrected-lifecycle-v6.html)
**Companion release:** v0.8.2 (description framework + compendium rewrite) — separate spec.
**Date:** 2026-05-13

## 1. Summary

v0.8.1 rewrites the addiction / withdrawal / tolerance / overdose lifecycle to match the v6 design. The shipped v0.8.0 engine has three structural bugs:

1. **Withdrawal auto-triggers on addiction.** The post-use failure cascade applies the Addiction AE *and* the Withdrawal AE in the same step, sets the actor's `withdrawal` flag map immediately, and starts the long-rest tick. v6 requires withdrawal onset to be its own event — only on a failed Abstain check followed by a failed Con save.
2. **Overdose has no gating.** Each use rolls d100 ≤ `chancePercent` unconditionally. v6 requires Tolerance Points to cross a per-substance Threshold before any overdose roll happens.
3. **Tolerance is a lockstep stack.** Current model is `tolerance.stacks: 1..N` with a `maxStacks` soft cap. v6 requires the dual **Count / Points** model that drives both Altered attenuation and Overdose threshold gating.

This spec defines the engine truth. v0.8.2 then rewrites compendium content + descriptions to match.

## 2. Goals

- Two-phase lifecycle: **On Use** (Phase 1) and **On Long Rest** (Phase 2), each with its own roll surface.
- Three independent saves/checks: **Addiction Save** (Con), **Abstain Check** (Wis), **Withdrawal Save** (Con).
- Tier-driven mechanical scaling on the standard 5e DC ladder.
- Author surface that hides internal math; surfaces only the levers an author actually decides.
- Per-block toggles preserved (`addiction.enabled` / `withdrawal.enabled` / `tolerance.enabled` / `overdose.enabled`); a substance can be addictive without withdrawal, OD-only without addiction, etc.
- Author-controlled symptom content: tier sets *likelihood* of withdrawal landing, not its severity. DC 30 substance can produce a mild rash; DC 5 can produce psychosis.

## 4. Concept changes vs v0.8.0

| Concept | v0.8.0 (now) | v0.8.1 (target) |
|---|---|---|
| Withdrawal Mod | numeric authored field driving DC + duration | **Tier 1-6**; hidden internal field. DC drives it via snapping. |
| Tier source | implicit | **Snapped from authored Withdrawal DC** on the {5,10,15,20,25,30} ladder. Nearest-by-distance; clamp at endpoints. |
| Withdrawal DC | derived `8 + mod` | **Hand-typed.** Engine snaps it to nearest tier to derive Rate / Threshold / Max Count. |
| Addiction DC | hand-typed | **Hand-typed** (unchanged). |
| Abstain | one-button "pick a substance to resist" dialog | **Per-substance opt-in Abstain** in a combined long-rest dialog. Unselected = consume normally. |
| Abstain ability | "save" via `rollSavingThrow` (bug) | **Wis ability check** via `rollAbilityCheck`. Hand-typed DC. |
| Tolerance | `stacks: 1..N`, `maxStacks` soft cap | **Count + Points** dual model. Count drives Altered attenuation; Points = Count × Rate drives Overdose threshold gating. |
| Tolerance decay | none | **On pass-Abstain or forced-Abstain**: Count -= decay (default 1). |
| Overdose trigger | unconditional d100 per use | **Threshold gate, then chance roll**: only if Points ≥ Threshold, roll d100 ≤ chancePercent. |
| Withdrawal duration | `restsRemaining` (long rests) | **Hand-typed real-time value + unit dropdown** (minutes/hours/days/weeks/months). Stored as `effect.duration.seconds`; Times-Up (bundled with DAE) handles expiry. |
| Withdrawal onset | applied with Addiction in same step | **Separate event**: only on failed Abstain → failed Withdrawal Save during long rest. |

## 5. Tier table

Author-facing surface: hand-typed **Withdrawal DC** + read-only previews. Tier is internal.

| Tier | Label | Nominal DC | Rate (Pts/Count) | Overdose Threshold (Pts) | Max Count | Uses → OD-eligible |
|---|---|---|---|---|---|---|
| 1 | Very Easy | 5 | 1 | 8 | 8 | 8 |
| 2 | Easy | 10 | 2 | 12 | 6 | 6 |
| 3 | Medium | 15 | 3 | 15 | 5 | 5 |
| 4 | Hard | 20 | 5 | 20 | 4 | 4 |
| 5 | Very Hard | 25 | 8 | 24 | 3 | 3 |
| 6 | Nearly Impossible | 30 | 13 | 26 | 2 | 2 |

Rate progression is Fibonacci `[1, 2, 3, 5, 8, 13]`.
Max Count = `ceil(Threshold / Rate)`.

### 5.1 DC → tier snapping

```
function snapDcToTier(dc) {
  const ladder = [5, 10, 15, 20, 25, 30];
  if (dc <= 5)  return 1;
  if (dc >= 30) return 6;
  // No integer DC ties on the ladder (boundary points 7.5, 12.5, … are non-integers).
  return 1 + indexOfClosest(ladder, dc);
}
```

Examples: DC 7 → tier 1; DC 12 → tier 2; DC 17 → tier 3; DC 23 → tier 5.

**The snap drives Rate / Threshold / Max Count only — it does NOT change the save DC.** The Withdrawal Save rolls against the authored DC (14 in the example), while the Tolerance / Overdose math runs off Tier 3's profile (Rate 3, Threshold 15, Max Count 5). Authoring a non-ladder DC like 14 means: "a substance with Tier 3 tolerance behaviour but a slightly easier withdrawal save." This is the override path.

## 7. Lifecycle: Phase 1 (on use)

Fires on `postUseActivity`. Pipeline, in order, gated by per-block `enabled` flags:

1. **Bypass consumption** — `consumeBypassIfAvailable(actor, substance)` (unchanged from v0.8.0; reads paraphernalia bypass AEs).
2. **Addiction Save** (Con, DC = `addiction.save.dc`).
   - Pass + already-addicted → Altered AE applies (subject to tolerance attenuation, §11).
   - Pass + not-addicted → Altered AE applies; no addiction.
   - Fail + already-addicted → Altered AE applies; addiction AE refreshed/extended.
   - Fail + not-addicted → **Addiction AE applies** (clone from `addiction.addictionEffectIds` or default). Altered AE also applies. **Withdrawal AE does NOT apply.**
3. **Tolerance increment** — if tolerance enabled, `count += 1` (clamped at `maxCount`). `points = count × rate`. Tolerance AE refreshed / re-applied.
4. **Overdose check** — if overdose enabled, run the overdose gate (§12).

Critical contract: **Phase 1 never applies a Withdrawal AE.** Withdrawal onset is exclusively a Phase 2 event.

## 8. Lifecycle: Phase 2 (long rest)

Fires on `dnd5e.preRestCompleted` (GM-arbitrated — only `game.users.activeGM === game.user` runs it).

### 8.1 Dialog (single combined)

For each substance the actor is currently addicted to:

```
┌─ Withdrawal Choices ────────────────────────────────────────────┐
│                                                                  │
│  You are addicted to the following substances and will           │
│  automatically consume one dose during the long rest unless you  │
│  specifically choose to abstain. Select all that apply;          │
│  unselected substances will be consumed as normal.               │
│                                                                  │
│  ☐ Coalshade Powder    Tol 3/9   Doses: 2 left                  │
│  ☐ Veilroot Tincture   Tol 1/2   Doses: 0 left  [forced Abstain]│
│                                                                  │
│                                              [Confirm]           │
└──────────────────────────────────────────────────────────────────┘
```

- **Unchecked = Use** (default). Engine forces-uses the substance via `activity.use({event: null}, {fastForward: true, chatMessage: true})` with the bypass flag set, exactly like today's "give in" failure path. Goes through the full Phase 1 chain.
- **Checked = Abstain** (opt-in). Engine runs Abstain Check (Wis, DC = `flag.withdrawal.abstain.dc`).
  - Pass → no withdrawal onset. Tolerance Count -= `decay`.
  - Fail → Withdrawal Save (Con, DC = `flag.withdrawal.dc` — the authored value, not the nominal tier DC). Pass → no withdrawal AE. Fail → Withdrawal AE applied (cloned from `withdrawal.effectIds` or default).
- **Forced Abstain** (no doses in inventory, or item missing) — same as Checked-Abstain but with the Wis check skipped entirely. Goes straight to Withdrawal Save. Tolerance Count -= `decay` regardless of save outcome.
- **Confirm** with nothing checked → all addictions use-normally. Closing the dialog without confirming is treated as Confirm-with-nothing-checked.

Forced-Abstain UI: render the row's checkbox as disabled-checked with a "no doses" tooltip; user can't uncheck it.

### 8.2 Tolerance decay timing

- Pass Wis Abstain → `count -= decay`
- Forced Abstain → `count -= decay`
- Fail Wis Abstain (engine forces use) → no decay; Phase 1 chain increments by 1
- Unchecked (player chose Use) → no decay; Phase 1 chain increments by 1

When `count` reaches 0, the Tolerance AE is removed; Points recalc to 0.

### 8.3 Withdrawal AE lifecycle

Applied with `effect.duration.seconds` set from the substance's authored `withdrawal.duration = { value, unit }`. Times-Up (bundled with DAE) handles expiry; no custom tick hook.

When Times-Up removes the AE, we listen on `deleteActiveEffect` to clear the actor's `flags.S&P.withdrawal[substanceId]` entry. (Active actor-flag is the canonical state; AE is the UI mirror, same as today.)

## 9. AE Change Keys (author surface)

These are the keys authors put on AE Change rows to affect engine math. All keys live under `flags.substances-and-paraphernalia.*`.

| Key | Type | Mode | Semantics |
|---|---|---|---|
| `flags.S&P.addiction.save.bonus` | number | ADD | Bonus to Addiction Save d20 total. |
| `flags.S&P.abstaining.check.bonus` | number | ADD | Bonus to Abstain Check d20 total. |
| `flags.S&P.withdrawal.save.bonus` | number | ADD | Bonus to Withdrawal Save d20 total. |
| `flags.S&P.tolerance.countModifier` | number | ADD | Adjusts Tolerance Count on read (e.g. paraphernalia that reduces effective Count). |
| `flags.S&P.overdose.chanceModifier` | number | ADD | Adjusts the per-use Overdose chance (clamped 0-100). |
| `flags.S&P.overdose.thresholdModifier` | number | ADD | Adjusts the Overdose Threshold (in Points). |
| `flags.S&P.vignetteColor` | string | OVERRIDE | Existing — withdrawal-screen vignette color. |
| `macro.tokenMagic` | string | CUSTOM | Existing — TMFX preset name. |

Authoring contract: read the *applied* values from `actor.flags.S&P.*` at roll time (Foundry resolves Change rows into actor flags by AE apply). The engine queries actor flags, not AE Change rows, when computing rolls.

## 10. aeRole enumeration

`flags.substances-and-paraphernalia.aeRole` is an authored string on AE templates. Used by `findEffectsByRole(actor, role)` for locale-independent AE lookup. Substring fallback (existing) stays as a warn-on-fallback safety net.

Roles: `altered`, `addiction`, `withdrawal`, `tolerance`, `overdose`, `bypass`.

Same naming substrings (existing AE naming contract in CLAUDE.md):
- altered AE name contains `altered`
- addiction AE name contains `addict`
- withdrawal AE name contains `withdraw`
- tolerance AE name contains `tolerance`
- overdose AE name contains `overdose`
- bypass AE name contains `bypass`

## 11. Tolerance attenuation (Altered AE scaling)

When applying the Altered AE, engine multiplies every numeric `value` on the AE's Change rows by an attenuation scalar.

**Default curve (halving array):** `[1.0, 0.5, 0.25, 0.125, 0]`, indexed by current Tolerance Count.

- Count 0 → 100% effect (the curve's leading 1.0 covers the no-tolerance case).
- Count 1 → 50%.
- Count 2 → 25%.
- Count 3 → 12.5%.
- Count ≥ 4 → 0% (Altered AE applied but all values zeroed out; effectively dormant).

When Max Count < array length (high tiers), the trailing array entries are simply unreachable. At Tier 6 (Max Count 2), the player only ever sees the first three values (`1.0`, `0.5`, `0.25`).

**Override:** substances may author `tolerance.attenuationCurve = [...]` on the substance flag block. Engine reads the substance's curve first, falls back to halving.

**Low-tier behaviour:** when Max Count exceeds the array length (Tiers 1-3, Max Count 5-8), Counts past the array's last index continue to return the last value (0). At Tier 1 (Max Count 8), Altered is fully suppressed from Count 4 through Count 8; the player can keep using the substance to walk Tolerance Points toward Overdose Threshold without further Altered effect. This is intentional — it models "chasing a high you can't reach anymore."

**Implementation note:** scaling happens at AE *clone* time (when engine applies the Altered AE to the actor). The cloned AE's `changes[].value` is multiplied scalar-then-stringified. This means re-applying the AE (e.g. via second Phase 1 cycle) recomputes the scalar from current Count.

## 12. Overdose mechanic

```
if (!overdose.enabled) return;
if (currentPoints < threshold + thresholdModifier) return;
const chance = clamp(0, 100, overdose.chancePercent + chanceModifier);
if (rollD100() <= chance) applyOverdoseAE();
```

Overdose AE templates listed in `overdose.effectIds`. If empty, engine builds a minimal default AE marked with `aeRole: "overdose"`.

## 14. Engine architecture

### 14.1 Pure helpers (testable, no Foundry globals)

- `scripts/data/tier-table.js` — `snapDcToTier(dc)`, `tierProfile(tier) → {dc, rate, threshold, maxCount}`.
- `scripts/data/tolerance.js` — `currentPoints(count, rate)`, `applyAttenuation(value, count, curve)`, `decayCount(count, decay)`.
- `scripts/data/overdose-gate.js` — `shouldRollOverdose(points, threshold, thresholdModifier)`, `rollOverdoseChance(rng, chancePercent, chanceModifier)`.
- `scripts/data/withdrawal-duration.js` — `durationToSeconds(value, unit)`.
- `scripts/data/abstain.js` — keep `defaultAbstainDc` for back-compat, mark deprecated; new Abstain DC is hand-typed.

### 14.2 Foundry-coupled hooks

- `scripts/hooks/addiction.js` — Phase 1 chain. **Stop applying Withdrawal AE here.** Stop setting `actor.flags.S&P.withdrawal[id]` in `applyOutcome`.
- `scripts/hooks/long-rest-abstain.js` — replace `promptCombinedAbstain` (button-per-substance) with a single combined dialog using `<input type="checkbox">` rows. Rewrite the post-confirm pipeline to walk each row and dispatch Use / Abstain / Forced-Abstain. Fix `rollAbstainSave` → `rollAbstainCheck` (`actor.rollAbilityCheck`).
- `scripts/hooks/tolerance-decay.js` — new. Subscribes to whatever event the long-rest dialog raises ("abstain resolved for substance X with outcome Y") and decays Count appropriately.
- `scripts/hooks/withdrawal-cleanup.js` — new. Listens on `deleteActiveEffect`; if the deleted AE has `aeRole: "withdrawal"` and a `sourceSubstanceId`, clear the matching actor-flag entry.

### 14.3 UI

- `scripts/ui/details-tab.js` — refactor Addiction fieldset per §6. Remove `withdrawalMod` input; add Withdrawal DC, Abstain DC, Withdrawal Duration (value + unit), Tolerance Decay. Tier preview is a live read-out under the Withdrawal DC field. Read-only previews for Rate/MaxCount/Threshold.
- `scripts/ui/abstain-dialog.js` — new. The combined dialog described in §8.1.

### 14.4 Settings

- Remove `voluntaryAbstainEnabled` (per CLAUDE.md: "don't ship a setting whose off-state nobody actually wants"). Long-rest dialog is baked in.

## 18. Open spec decisions (defaulted; flip on review)

These were not explicitly asked-and-answered during brainstorming; defaults below. Override during spec review if any are wrong.

- **D1**: Tolerance state map at actor uses `lastIncrementedAt` + `lastDecayedAt` (both ISO timestamps). Drops "decay if not used in N hours" as a feature; symmetry-only.
- **D2**: Forced-Abstain chat card reads `"{actor} has no doses of {item}; withdrawal onset save vs DC {n}…"`. Make this localizable as `FISHUT.Phase2.ForcedAbstain.Intro`.
- **D3**: Dialog row sorting — alphabetical by substance name. Stable per-rest.
- **D4**: When the Altered AE is reapplied with a different attenuation scalar (Count changed since last use), engine deletes the prior Altered AE first to avoid stacking. (Same pattern current Tolerance code uses.)
- **D5**: `validate-content.mjs` treats missing `withdrawal.dc` as an error on `kind: "substance"` items where `addiction.enabled !== false`. Authors who want a substance with addiction but no withdrawal-survival path set `withdrawal.enabled: false` explicitly.
- **D6**: Overdose `chancePercent` field range stays `0-100` integer. `chanceModifier` from AE Changes can drive effective chance above 100 (cap at 100) or below 0 (cap at 0) — clamp at roll time, not at modifier read.
