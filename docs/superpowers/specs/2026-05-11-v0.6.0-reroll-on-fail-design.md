# v0.6.0 — Reroll-on-fail bypass tier

**Status:** Approved 2026-05-11
**Thesis:** Add a fourth save-bypass tier — `reroll-on-fail` — to the AE-flag modifier pipeline. Honor SPEC's single-thesis-per-release discipline; defer everything else.

- Resolution composition: tier-ranked between `auto-pass` and `advantage` (rationale below).
- Save flow: when `reroll-on-fail` wins, roll once; if the roll fails the DC, roll a second time and use that result. No advantage/bonus is layered.

## 3. Resolution composition

New `TIER_RANK`:

```js
const TIER_RANK = Object.freeze({
  "auto-pass": 0,
  "reroll-on-fail": 1,
  advantage: 2,
  "+N": 3,
});
```

Rationale: for a flat d20 vs flat DC the two are mathematically identical (`P(pass) = 1 − q²` for both, where `q` is single-roll fail prob), but reroll-on-fail interacts more strongly with downstream dnd5e features — Halfling Lucky, Bardic Inspiration, ability-check bonuses applied per roll — and the user's stated authoring intent is that reroll be the rarer/stronger tier in narrative weight. Whatever the per-table math, the pipeline collapses to a single winning tier, so authoring sources don't have to think about composition.

`pickBypassResolution` gets a new branch after the auto-pass branch and before the advantage branch. Like auto-pass and advantage, it is **single-pick**: filter eligible candidates by admin + kind + type + uses, sort ascending by AE id, take the first, return `{ resolution: "reroll-on-fail", sources: [winner], bonus: 0 }`. Multiple reroll AEs do not stack — second-strongest contributors are ignored, same as auto-pass/advantage today.

Tie-break semantics across tiers are unchanged: strictly compare `TIER_RANK[a] < TIER_RANK[b]`.

## 4. Save flow

`scripts/hooks/addiction.js#rollSave(actor, ability, dc, opts)` accepts an additional opt:

```js
{ advantage?: boolean, bonus?: number, reroll?: boolean }
```

When `reroll === true`:
1. Roll once via `actor.rollSavingThrow ?? actor.rollAbilitySave`, **with no advantage and no bonus** (reroll wins outright over `advantage`/`+N`, so they cannot co-fire).
2. If `total >= dc`, return that roll.
3. Else roll a second time with identical config (no advantage, no bonus) and return the second roll.

Only the **final** roll is the canonical result for chat / addiction-AE application.

`applyOutcome` routes the chat key on `outcome.modifier.resolution === "reroll-on-fail"`:
- `FISHUT.Addiction.Save.PassWithReroll` — second roll succeeded (or first roll succeeded and no reroll was needed; we still surface that the reroll bypass was eligible, so the player sees why).
- `FISHUT.Addiction.Save.FailWithReroll` — both rolls failed and withdrawal lands.
- `FISHUT.Addiction.Save.FailNoWithdrawalWithReroll` — both rolls failed but the substance has no withdrawal hook (parallels the existing `FailNoWithdrawal` key).

Pass/fail outcomes route through the same downstream code paths as today — only the chat string differs.

## 5. Use consumption

Per SPEC §"usesPerDay semantics" (line 233-234): `system.uses.spent` ticks **once per consumption attempt**, not per re-roll. The existing decrement loop in `consumeBypassIfAvailable` already runs once per winning AE — no change needed. The reroll branch in `rollSave` runs entirely downstream of the bypass-consumption step, so the second die does not trigger a second decrement.

This means a paraphernalia with `usesPerDay: 1` and `reroll-on-fail` grants one chance-with-safety-net per day — both dice cost the same single use.

## 6. Schema cleanup

In the same v0.6.0 release, delete the dead-end `addictionSaveBypassTypes` array from `scripts/data/schema.json`:

```json
"addictionSaveBypassTypes": [
  { "id": "auto-pass", "labelKey": "FISHUT.SaveBypass.Type.AutoPass" }
],
```

It is a v0.2-era leftover; nothing reads it post-v0.3 (the canonical list is `modifier.types`). Removing it now avoids the schema growing a stale alternate enum next to the live one. Pre-1.0, no migration shim.

## 8. Tests

New unit file `test/unit/modifier-resolution-reroll.test.mjs` (**add to `package.json` `test:unit` script** — it's an explicit file list, not a glob):

- Reroll wins over advantage when both are eligible.
- Auto-pass beats reroll when both are eligible.
- Reroll filtered out when `appliesTo` doesn't match the substance admin.
- Reroll filtered out when `usesPerDay` is configured but no uses remain.
- Within-tier: two reroll candidates → ascending-id tie-break picks one; second AE not in `sources`.

Extend `test/unit/modifier-resolution.test.mjs` with cross-tier cases proving the new ordering across all four tiers.

New Quench test in `test/quench/test-suite.mjs` (`S&P · Reroll-on-fail save flow`):
- Build an actor with a paraphernalia granting `reroll-on-fail` + `usesPerDay: 1`.
- Trigger a `postUseActivity` on a substance whose DC is high enough that the first die almost certainly fails (stub the d20 via the dnd5e/Foundry hook surface that the existing Quench tests already use).
- Assert: two dice rolled, one `system.uses.spent` increment, chat key matches the expected pass/fail variant.
